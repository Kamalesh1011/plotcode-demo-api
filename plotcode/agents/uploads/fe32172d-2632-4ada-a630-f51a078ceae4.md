## 1. Overview

**App Studio** is an AI-powered full-stack application development platform that enables users to build, deploy, and manage web applications through natural language interactions. The platform combines multiple AI agents, real-time code editing, live preview, and automated deployment capabilities.

### Key Features

- **AI-Powered Development**: Multiple specialized AI agents for different aspects of development
- **Real-Time Collaboration**: Live code editing with AI assistance
- **Visual Preview**: Instant preview of generated applications
- **Multi-Framework Support**: React.js, Next.js, and other modern frameworks
- **Automated Deployment**: Seamless deployment to Fly.io and cloud platforms
- **Version Control**: Git integration and codebase versioning
- **Document Generation**: Automated PRD, BRD, Architecture, and Test Cases
- **Knowledge Base**: Context-aware development using uploaded documents

### System Capabilities

1. **Natural Language to Code**: Convert requirements into working applications
2. **Intelligent Code Generation**: Context-aware code generation with multi-agent coordination
3. **Live Application Preview**: Real-time application preview with error tracking
4. **Automated Testing**: Generate and execute test cases
5. **Cloud Deployment**: One-click deployment to multiple cloud providers
6. **Version Management**: Track changes and revert to previous versions

## 2. System Architecture

App Studio consists of three main backend services and a React-based frontend:

```

│                        User Interface                        │
│              (React Frontend - appbuilder)                   │
└─────────────────┬───────────────────────────────────────────┘
                  │

┌─────────────────────────────────────────────────────────────┐
│                  API Gateway / Router                        │
│                  (Nginx Reverse Proxy)                       │
└─────────────────┬───────────────────────────────────────────┘
                  │
        ┌─────────┴─────────┬──────────────┐
        ▼                   ▼              ▼
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│  App Studio  │   │  Basic Fly   │   │ VM Reverse   │
│   Backend    │   │     App      │   │    Proxy     │
│  (Go/Echo)   │   │  (Go/Echo)   │   │  (Go/HTTP)   │
└──────┬───────┘   └──────┬───────┘   └──────┬───────┘
       │                  │                   │
       └──────────┬───────┴───────────────────┘
                  ▼
         ┌─────────────────┐
         │    MongoDB      │
         │  (Data Store)   │
         └─────────────────┘
```

### Service Descriptions

#### 1. **App Studio Backend** (`apps/app-studio`)
- **Port**: Configurable (default: varies)
- **Framework**: Go + Echo
- **Purpose**: Core AI orchestration, app management, and API endpoints

**Key Responsibilities**:
- AI agent coordination (Master, Code, Architecture, Testing agents)
- Application CRUD operations
- Chat history management
- Document generation (PRD, BRD, Architecture)
- Integration with external services (GitHub, Figma, Supabase)
- Cloud build orchestration
- Deployment management

**Key Responsibilities**:
- File system operations for user apps
- PM2 process management
- Build and transfer operations
- Application health checks
- Log streaming
- Command execution in app context


#### 3. **VM Reverse Proxy** (`apps/vm-reverse-proxy`)
- **Port**: Configurable (default: varies)
- **Framework**: Go + HTTP
- **Purpose**: Dynamic routing to user application VMs

**Key Responsibilities**:
- Domain-to-VM mapping
- VM assignment with LRFU (Least Recently/Frequently Used)
- Cache management for routing
- Proxy requests to appropriate Fly.io VMs
- Health monitoring


#### 4. **Frontend - appbuilder** (`libs/appbuilder`)
- **Framework**: React + TypeScript
- **Build Tool**: Vite
- **Purpose**: User interface for App Studio

**Key Components**:
- **Builder**: Main IDE interface
- **Chat**: AI chat interface with streaming
- **Editor**: Monaco-based code editor
- **Preview**: Live application preview
- **Architecture/BRD/TestCases**: Document viewers
- **Settings**: App configuration

### 4.2 Proxy and VM Assignment Workflow

Based on the second workflow diagram:

```
┌──────────────────────────────────────────────────────────┐
│                    USER (Browser)                         │
└────────────────────────┬─────────────────────────────────┘
                         │
                         │ preview--appname-unique.vanijstaging.adya.ai
                         │
                         ▼
            ┌────────────────────────┐
            │   Nginx Proxy          │
            │   vanijstaging.adya.ai │
            └────────┬───────────────┘
                     │
                     │ Forward request
                     │
                     ▼
            ┌────────────────────────┐
            │   VM Reverse Proxy     │
            │   (Routing Service)    │
            └────────┬───────────────┘
                     │
        ┌────────────┼────────────┐
        │            │            │
        ▼            ▼            ▼
   ┌─────────┐ ┌─────────┐ ┌─────────┐
   │  Cache  │ │   DB    │ │  Assign │
   │  Miss   │ │  Hit    │ │   VM    │
   └────┬────┘ └────┬────┘ └────┬────┘
        │           │           │
        │           │           │ LRFU Algorithm
        │           │           │
        └───────────┼───────────┘
                    │
                    ▼
        ┌───────────────────────────┐
        │   Setup VM if not there   │
        │   (Assign to Cache & DB)  │
        └────────┬──────────────────┘
                 │
                 ▼
        ┌───────────────────────────┐
        │   Fly.io VM               │
        │   app.fly.dev             │
        └────────┬──────────────────┘
                 │
                 │ If cache miss,
                 │ setup files
                 │
                 ▼
        ┌───────────────────────────┐
        │   Proxy to Fly VM         │
        │   Return response         │
        └───────────────────────────┘
```

**Detailed Flow**:

1. **User Access**:
   - User navigates to `preview--appname-unique.vanijstaging.adya.ai`
   - Browser sends HTTP request

2. **Nginx Routing**:
   - Nginx receives request
   - Routes to VM Reverse Proxy service

3. **Routing Service Logic**:
   ```
   IF app_url in cache:
       fly_url = cache.get(app_url)
   ELSE IF app_url in database:
       fly_url = database.query(app_url)
       cache.set(app_url, fly_url)
   ELSE:
       fly_url = assign_vm_with_lrfu()
       database.save(app_url, fly_url)
       cache.set(app_url, fly_url)
   ```

4. **VM Assignment (LRFU)**:
   - Query all VMs and their current assignments
   - Calculate LRFU score for each VM
   - Select VM with lowest score
   - Create new assignment

5. **Setup (if new assignment)**:
   - Call App Studio API to setup files
   - Stream files to Fly.io VM
   - Install dependencies
   - Start application

6. **Proxy Request**:
   - Forward request to Fly.io VM
   - Stream response back to user
   - Handle errors gracefully

## 11. Performance Optimization

### Backend Optimizations

1. **Caching**:
   - Domain-to-VM mappings cached (5 min TTL)
   - LLM responses cached where appropriate
   - MongoDB query result caching

2. **Connection Pooling**:
   - MongoDB connection pool
   - HTTP client reuse

3. **Concurrency**:
   - Singleflight for duplicate requests
   - Goroutines for parallel operations

### Frontend Optimizations

1. **Code Splitting**:
   - Lazy loading of components
   - Route-based splitting

2. **Memoization**:
   - React.memo for expensive components
   - useMemo/useCallback hooks

3. **Debouncing**:
   - File edit updates
   - Search inputs

4. **Virtual Scrolling** (if needed for large file trees)
